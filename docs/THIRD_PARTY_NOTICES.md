# Third-Party Notices

This project uses third-party open source packages through npm, Cargo, and pip.
Those packages are not vendored as source in this repository, except for
generated Tauri schema files under `src-tauri/gen/schemas/`.

## Compliance Rule

When publishing source, binary builds, installers, archives, or app bundles:

- keep this project license in `LICENSE`;
- keep upstream copyright and license notices for third-party code that is
  copied, bundled, linked, or otherwise redistributed;
- include dependency license notices in release artifacts, especially desktop
  bundles and self-use packages;
- do not copy code from another repository into this repository unless its
  license header and attribution are preserved.

MIT, ISC, and BSD-style licenses are permissive, but they still require the
copyright notice and permission/license notice to stay with redistributed copies
or substantial portions of the software.

## Current Lockfile Audit

Run:

```bash
npm run license:audit
```

The current npm lockfile contains only packages with declared licenses. The
current Cargo dependency graph contains declared licenses as well. Some
transitive dependencies use MPL-2.0, Unicode-3.0, CC-BY-4.0, or similar
notice-bearing licenses; keep their notices in binary/app distributions.

## Generated Files

The files in `src-tauri/gen/schemas/` are generated by the Tauri toolchain.
Treat them as Tauri-derived generated material and retain Tauri license notices
when redistributing bundles that include them.

## Release Checklist

- Run `npm run license:audit`.
- Regenerate or review third-party notices after dependency updates.
- Include third-party notices in release archives and desktop app bundles.
- Re-check dependency licenses before changing repository visibility or
  publishing packages.
